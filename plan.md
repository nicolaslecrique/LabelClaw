Build the following APP

## Purpose

This is an app to label a dataset, where the user can create it's own labelling UI

## Coding style

* Always use the most standard solution as of 2026, This is true for folders layout, tools, naming, dependencies, patterns, APIs...
* Always respect KISS and YAGNI principles
* Always Respect Separation of concerns, SRP, uncoupling and cohesion concepts.
* Prefer functional stype programming (pure functions, no side-effects, immutability, precise types to enforce invariants)
* Use moderns and standards linters, type checkers and formatters, always use a standard but strict configuration
* Add automated end-to-end tests for all specifications

## Backend Technical specifications

* Coded in Golang
* call the LLM (from any provider)
* serve the frontend files
* Save the configuration in json files
* Expose a Rest API that can be called by the frontend or an external service
* handle the business logic

## Frontend

* Coded in typescript
* Only interact with the golang backend RestAPI
* Only pure UI logic

## Business Specifications

The app is composed of one "configuration" tab, and one "labelling" tab

### Configuration tab

The configuration tab expose a left panel that contains 4 rows: 3 text field and a "generate labelling panel" button

* "sample JSON Schema": a text field so the user can write a json schema that represent the labelling task input
* "label JSON Schema": a text field so the user can write a json schema that represent the labelling task output
* "UI prompt": a text field so the user can describe the UI of a React Component that will be generated by the llm (through the backend) and rendered dynamically

When the user click the button, a react component is generated (by llm) and renderer on the right panel with a sample data

The generated React component takes data instance matching the "Sample Json Schema" and expose a function to return an output matching the label JSON Schema

* On top of the right panel, there is a "save labelling panel" button, that send the save the generated code in a configuration file (in the backend)
